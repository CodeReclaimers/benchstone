# Benchstone — project-specific instructions

## Progress logging

This project does **not** use local `progress-YYYYMMDD.md` files. Record session
updates via the refstore MCP tool `mcp__refstore__write_session_note` instead.
The cadence and content rules from the global `CLAUDE.md` still apply (record on
each commit or significant change; include summary, decisions with rejected
alternatives, verification evidence verbatim, what was intentionally not done,
and next steps).
